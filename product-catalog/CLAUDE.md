# Product Catalog: Polished Artifacts

## Purpose

This folder contains **finished, shareable work**. Documents here should be ready for team consumption, stakeholder presentations, or external sharing.

## What Goes Here

- Team charters and mission statements
- Strategy documents and roadmaps
- Decision briefs and PRDs
- Stakeholder presentations
- Published analysis and recommendations

## Quality Bar

Before adding a document here, verify:

- [ ] **Accurate** - Claims match reality (code, data)
- [ ] **Clear** - Can be understood without context
- [ ] **Owned** - You can defend every sentence
- [ ] **Evidenced** - Key claims have links/references
- [ ] **Audience-appropriate** - Right level of detail

## Folder Structure (Suggested)

```
product-catalog/
├── strategy/        # Team identity, mission, value proposition
├── planning/        # Roadmaps, quarterly plans, priorities
├── decisions/       # Decision briefs, PRDs, RFCs
├── communications/  # Stakeholder updates, presentations
└── _examples/       # Example templates (don't edit)
```

## When Helping

- Maintain high quality standards
- Preserve formatting consistency
- Suggest improvements to clarity
- Flag unsupported claims
- Keep audience in mind

## Document Lifecycle

1. Draft in `sandbox/`
2. Polish and review
3. Move to `product-catalog/`
4. Share with intended audience
5. Update as needed, note version history
