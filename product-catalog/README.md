# Product Catalog

Polished artifacts ready for team and stakeholder consumption.

## Purpose

This is where finished work lives. Documents here should be:

- **Complete** - Not works-in-progress
- **Clear** - Understandable without additional context
- **Accurate** - Claims backed by evidence
- **Shareable** - Ready for intended audience

## Suggested Structure

```
product-catalog/
├── strategy/        # Team identity, mission, value proposition
├── planning/        # Roadmaps, quarterly plans
├── decisions/       # Decision briefs, PRDs, RFCs
├── communications/  # Stakeholder updates, presentations
└── _examples/       # Example templates
```

## Workflow

1. **Draft** in `sandbox/`
2. **Polish** with AI assistance
3. **Review** against quality checklist
4. **Move** to appropriate subfolder here
5. **Share** with intended audience

## Quality Checklist

Before adding a document:

- [ ] Accurate - Claims match reality
- [ ] Clear - Standalone readability
- [ ] Owned - You can defend every sentence
- [ ] Evidenced - Key claims have references
- [ ] Appropriate - Right detail level for audience
